Boids Demo

A boid refers to a bird like object. There are several approaches that can be taken to achieve a bird like flocking behavoir. Generally boid algorithms start with the same set of rules, which are.

  • > Boids should follow adjacent boids
  • > Boids should avoid adjacent boids
  • > Boids should follow the flock

The great thing about this design is that it supports custom behavoirs easily. For example, if you want the flock to orbit around or move towards a point then you can make a rule that alters each boids velocity to get the desired behavoir. In this demo, each boid has a ray cast in the direction it is currently moving, if that ray collides with any boid then the boid adpats it's own velocity based on the boid rules.
